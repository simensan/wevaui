### Added
- `TMP_FontAsset` as an authoritative SDF text source (`Runtime/Text/Tmp/TmpFontAssetSource.cs`, `Runtime/Text/Tmp/TmpFontAssetRegistry.cs`, `Runtime/Text/Tmp/TmpFontMetrics.cs`). Authors generate a TMP font asset via `Window > TextMeshPro > Font Asset Creator` (SDFAA recommended) and register it at app startup with `TmpFontAssetRegistry.RegisterFontAsset("sans-serif", asset)`. `SdfBootstrap.PickBest` then returns a `TmpFontMetrics` (implementing the same `IFontMetrics` + `IGlyphMetrics` surface as `SdfFontMetrics`) so layout, baking and rendering paths are unchanged — only the glyph data source flips. `TmpFontAssetSource` reads `characterTable` -> glyph index -> `glyphLookupTable` to produce our `(GlyphRect, GlyphMetrics)` pair, converting atlas-pixel rects to 0..1 UVs against the asset's atlas dimensions and scaling font-unit metrics by `(fontSize / faceInfo.pointSize)`. Kerning is read from `fontFeatureTable.glyphPairAdjustmentRecords` and exposed via `GetKern(left, right, fontSize)`. `SdfGlyphAtlasAdapter.TryShape` short-circuits through a TMP-specific shaper when `TmpSource` is set, emitting `SdfGlyphQuads` directly from the pre-baked atlas; missing codepoints fall through to the existing `SdfGlyphRasterizer` path so unsupported glyphs still draw. `GlyphAtlas.TextureOverride` lets the existing `AtlasRegistry` -> `GetTextureById` plumbing return the TMP atlas Texture2D without standing up a parallel registry. We do NOT instantiate any `TMP_Text` MonoBehaviour — TMP is treated as a glyph-data backend only. Empty registry => transparent fall-through to the `FontEngine` SDF path; opt-in only. Tests: `TmpFontAssetSourceTests` (8), `TmpFontMetricsTests` (7), `TmpBootstrapIntegrationTests` (3).
- `UICssDiagnostics` (`Runtime/Diagnostics/UICssDiagnostics.cs`) — central non-fatal-warning channel for the CSS / HTML engine. Routes silent-skip paths (unsupported property, unparseable value, no-op filter chain, unknown @-rule, calc() in gradient stop, unrecognised @media feature, `:not(complex)`, `@import`, `clip-path`/`mask` stubs) through `Debug.LogWarning` with a consistent `[Weva/CSS] {source}: {detail}` prefix. Each (source, detail) pair is deduplicated for the process lifetime so hot paths don't flood the console. Gated on `UNITY_EDITOR || DEVELOPMENT_BUILD`; release builds compile to a no-op. Wire-up sites: `CssParser.SkipUnknownAtRule`, `CssParser.ParseImportRule`, `ComputedStyle.Set` (unknown property + stub property), `CssValue.TryParse` (parse failure), `URPRenderBackend.Submit(PushFilterCommand)`, `BoxToPaintConverter.EmitDecorationsLocal` (combined filter chain), `BackgroundResolver.TryParseStop` (calc() stop position), `MediaFeatureQuery.Evaluate` (unrecognised feature), `SelectorParser.ParseSimpleForNot` (`:not(complex)`). New `clip-path` / `mask` stub registration in `CssProperties` so they cascade-roundtrip while flagging their no-op status. Tests: `UICssDiagnosticsTests` (4 cases — unknown-property emission, custom-property no-emit, dedupe contract, disabled bypass).
- Per-layer `<position> / <size>` clipping for gradient backgrounds (`Runtime/Paint/Brush.cs`, `Runtime/Paint/Conversion/BackgroundResolver.cs`, `Runtime/Rendering/URP/UIBatcher.cs`, `Runtime/Rendering/Shaders/Weva-Quad.shader`). `Brush.Gradient` now accepts an optional `BackgroundTile` (origin + tile size + per-axis repeat/gap) — the renderer clips the gradient to that tile rect and discards fragments outside it (no-repeat) or wraps modulo `tile + gap` (repeat / `space`). `BackgroundResolver` builds the tile from the cascade-resolved per-layer position/size/repeat for gradient layers (matching the existing image-layer machinery) and re-parses the gradient in tile-local bounds so radial/conic centers + radii resolve against the tile, not the full box. `UIBatcher.EmitGradient` packs tile origin/size into instance slot 4 (BorderWidths) and no-repeat flags + gap into slot 9 (BorderStyles); both slots are otherwise unused for non-bordered gradients. The shader's linear/radial/conic branches share a `Weva_GradientTileUv` helper that mirrors the `_BRUSH_IMAGE` tiling path. Default tile (gradient with no per-layer rect) = origin (0,0), size = box, no-repeat — preserves the legacy "fill the whole quad" rendering for unmodified gradients. Fixes the randhtml `.world::before` mountain triangles and other multi-layer gradient stacks like `linear-gradient(135deg, transparent 50%, #1c2030 50%) 0 0 / 30vmin 30vmin no-repeat, ...`. Tests: `BackgroundMultiLayerTests.Gradient_layer_with_per_layer_position_size_carries_tile`, `UIBatcherTests.Gradient_brush_with_tile_packs_origin_size_and_norepeat_flags` + `Gradient_brush_without_tile_defaults_to_full_box_fill`.
- `border-image` 9-slice (`Runtime/Paint/Conversion/BorderImageResolver.cs`). New longhands registered: `border-image-source` (url/none), `border-image-slice` (1–4 lengths/percentages, optional trailing `fill` keyword tolerated), `border-image-repeat` (stretch/repeat/round/space, 1–2 values). Resolver splits the source into 9 atlas-UV sub-regions and maps them onto 9 box-local dest regions: 4 corners stretched to the corresponding `border-{side}-width`, 4 edges either stretched (`stretch`) or tiled along their axis (`repeat`/`round`/`space`), center deliberately omitted in v1 (matches CSS default — `fill` keyword honoring deferred). Each part renders as its own `FillRectCommand` with its own `Brush.Image` carrying the sub-rect + tile mode. Border widths come from the regular `border-{side}-width` properties; `border-image-width` and `border-image-outset` are not yet honoured. Tests: `BorderImageResolverTests` (13).
- `background-clip` / `background-origin` honoring (`Runtime/Paint/Conversion/BackgroundClipOrigin.cs`). Resolves the `border-box` / `padding-box` / `content-box` keywords into two box-local rects: a paint rect (the `FillRectCommand` bounds, default border-box) and an origin rect (reference for `background-position`, default padding-box). `BackgroundResolver.ResolveBackgroundLayersInto` accepts both and adjusts the `BackgroundTile.OriginX/Y` so the renderer's local-coords math is unchanged. Tests: `BackgroundClipOriginTests` (8).
- `space` and `round` repeat modes fully honoured. `round` adjusts the tile size so a whole number of repetitions fits the container exactly (`tile = box / round(box/tile)`). `space` distributes leftover pixels as gaps between tiles (`gap = (box - count*tile) / (count-1)`); the first/last tile sit flush against the container edges and the parsed `background-position` is overridden on each axis where space resolves. New `BackgroundTile.GapX/GapY` fields ferry the gap to the renderer; URP shader treats `stride = tile + gap` and discards fragments inside the gap region; IMGUI loop steps by stride and natural gap-skipping falls out of the inner-rect clip.
- `background-position` / `background-size` / `background-repeat` honoring (`Runtime/Paint/Conversion/BackgroundLayoutResolver.cs`, `Runtime/Paint/BackgroundTile.cs`). Resolves CSS Backgrounds 3 §3 values into a `BackgroundTile` with absolute pixel dimensions, origin offset, and per-axis repeat mode. Size: `auto` (intrinsic), `cover`, `contain`, `<length>{1,2}`, `<percentage>{1,2}` — with intrinsic-aspect-ratio derivation when one axis is `auto`. Position: keyword combos (top/right/bottom/left/center, with vertical/horizontal swapping), lengths, percentages — resolved against `(box - tile)` so `right` means flush-right. Repeat: `repeat`/`no-repeat`/`repeat-x`/`repeat-y` plus two-value form; `space`/`round` parse distinctly but render as `repeat` in v1. `Brush.Image` carries an optional `BackgroundTile` (null = legacy stretch). Renderer: URP shader's `_BRUSH_IMAGE` branch reads tile params from instance data and tiles with `frac()` / discards out-of-tile fragments for `no-repeat`. IMGUI fallback loops `GUI.DrawTextureWithTexCoords` per tile with sub-rect UV clipping at box edges. `BackgroundResolver.ResolveBackgroundLayersInto` accepts an optional `IImageRegistry` + `LengthContext` and queries the source's intrinsic dimensions before resolving. `BoxToPaintConverter.ImageRegistry` setter wires it through. Tests: `BackgroundLayoutResolverTests` (20).
- `<img>` natural-size resolution (`Runtime/Layout/BoxBuilder.cs`). HTML `<img width="N" height="N">` attributes resolve as CSS pixel lengths; when both CSS and HTML attrs are absent, BoxBuilder queries `LayoutEngine.ImageRegistry` for the source's intrinsic `Width`/`Height` and writes them into the ComputedStyle so layout treats `<img>` like any explicitly-sized element. Mixed cases (HTML width only, CSS height only) fill the missing axis from the registry. `LayoutEngine.ImageRegistry` setter; `BoxBuilder` constructor accepts the registry. Tests: `ImgIntrinsicSizeTests` (10).
- `AddressablesImageRegistry` (`Runtime/Paint/Images/AddressablesImageRegistry.Unity.cs`). Async `IImageRegistry` over Unity Addressables. `PreloadAsync(handles)` for warmup; `TryResolve` returns false until the load completes (back-end paints magenta until ready). Tries `Sprite` then `Texture2D`; caches both successes and failures. Gated by `WEVA_ADDRESSABLES` (added to `versionDefines` for `com.unity.addressables ≥ 1.21.0`); the headless build skips it.
- Unity-bound image adapters + renderer wiring (`Runtime/Paint/Images/*.Unity.cs`, `Runtime/Rendering/URP/UIImageAtlasRegistry.cs`, `Runtime/Rendering/URP/UIBatcher.cs`, `Runtime/Rendering/Shaders/Weva-Quad.shader`, `Runtime/Rendering/IMGUIDocumentRenderer.cs`). New types (all `#if UNITY_5_3_OR_NEWER` so the headless build skips them): `Texture2DImageSource` wrapping a `Texture2D`, `SpriteImageSource` wrapping a `Sprite` (atlas-aware UV rect from `sprite.textureRect`), and `ResourcesImageRegistry` — `Resources.Load<Sprite>` then `Resources.Load<Texture2D>` fallback with per-handle caching including negative cache. **IMGUI backend**: `IMGUIBackend.ImageRegistry` setter consumed by `Submit(FillRectCommand)`; resolved images render via `GUI.DrawTextureWithTexCoords` honouring sub-region UVs and a per-call `FilterMode.Point`/`Bilinear` swap from `Brush.ImageRendering`. Unresolved handles paint magenta. **URP batched backend**: `UIBatcher.ImageRegistry` resolves the handle, registers the texture with `UIImageAtlasRegistry` (mirrors `AtlasRegistry` for SDF text — int id per Texture2D), packs source-rect UV into `BrushParams.yz` + UV scale into `BorderColorTop.xy`. `UIRenderGraphPass` binds the texture to `_GlyphAtlas` when the batch's brush kind is `Image`, applies point/bilinear `filterMode` per `Brush.ImageRendering`. New `_BRUSH_IMAGE` shader keyword + matching SampleBrush branch in `Weva-Quad.shader` samples the bound texture with the per-instance source-rect UV. Game-code wiring: `imguiRenderer.ImageRegistry = myReg;` (or, for URP, set `UIBatcher.ImageRegistry` on the active batcher) — author writes `<img src="ui/heart">` and the registry maps that handle to a `Texture2D` / `Sprite`. v1 simplifications: source-rect is full image until `background-position`/`background-size` lands; URP path requires shader recompile when `_BRUSH_IMAGE` is added to the keyword stripping config; legacy `URPRenderBackend.cs` (mesh-builder, pre-batched) keeps the magenta fallback — only the production batched path is fully wired.
- Image rendering pipeline: HTML `<img src="handle">` and CSS `background-image: url("handle")` now flow into a `Brush.Image(handle, sourceRect, ImageRenderingMode)` paint command. New types: `IImageSource` (backend-agnostic texture handle, `Runtime/Paint/Images/IImageSource.cs`), `IImageRegistry` (`TryResolve(handle, out source)` lookup), and a default `InMemoryImageRegistry` impl with `Register`/`Unregister`/`Clear`. Authors write logical handles (`"ui/heart-icon"`) — game code populates the registry at startup with backend texture wrappers; the converter doesn't resolve handles, only ferries the string through the paint list. `BackgroundResolver.TryParseUrl` extracts the handle from `url(x)` / `url("x")` / `url('x')` (single + double quotes); `BackgroundResolver.ResolveBackground` and `ResolveBackgroundLayersInto` emit `Brush.Image` for url() segments. `BoxToPaintConverter` paints `<img>` between background and border in the box rect — `border-radius` clips the image, and `image-rendering` propagates so pixel-art icons get nearest-neighbor sampling. v1 simplification: source rect is `(0,0,1,1)` (full image); `background-position` / `background-size` / `background-repeat` honoring is a follow-up. `<img>` requires explicit width/height (CSS or HTML attribute) — natural-size resolution from `IImageSource.Width/Height` will land when layout consults the registry. Tests: `ImageRegistryTests` (10), `BackgroundUrlTests` (11), `ImgPaintTests` (7).
- `DirectionalNavigation` (`Runtime/Events/DirectionalNavigation.cs`) — spatial-focus picker for D-pad / arrow-key driven UIs. Game code wires the D-pad to `MoveFocus(NavDirection.Up/Down/Left/Right)`; the class finds the nearest focusable element in that half-plane using a weighted distance score (`parallel + perpendicular * PerpendicularPenalty`, default 2.0) so off-axis candidates lose to aligned ones — matches Smart-TV / console user expectations. No-current-focus fallback picks the topmost-leftmost focusable so the first D-pad press lights up something. Accepts a `Func<Element, NavRect?>` rect lookup so callers wire it to whatever their layout pipeline produces (the framework's `Box` X/Y/W/H, or a custom focus-ring source); returning null excludes the element. Public `NavRect` value type with center/right/bottom helpers. Reuses internal `FocusManager` for focusability and `EventDispatcher.Focus` for the transition. Tests: `DirectionalNavigationTests` (13).
- `VirtualPointerSource` (`Runtime/Events/VirtualPointerSource.cs`) — software-driven pointer that game code feeds gamepad / D-pad / scripted input into. Forwards into the same `EventDispatcher.DispatchPointer*` calls a hardware mouse would, so existing `:hover` / `:active` / `:focus-visible` cascade hooks fire correctly for controller-only UIs (console games, VR menus). API: `MoveTo`/`Move` (absolute & relative), `SetViewport` (clamping), `ButtonDown`/`ButtonUp`/`Click` (button mask is a bitfield), `AdvanceFromStick(stickX, stickY, pixelsPerSecond, deltaSeconds)` for the common gamepad pattern, `Modifiers` for combined keyboard chord state. Click synthesis still flows through `EventDispatcher`'s down/up matching — drifting off the target before release correctly suppresses the click. Zero per-frame allocations in steady state. Tests: `VirtualPointerSourceTests` (12).
- `image-rendering` property (`Runtime/Paint/ImageRenderingMode.cs` + `Runtime/Paint/Conversion/ImageRenderingResolver.cs`) — resolves to `Auto`/`CrispEdges`/`Pixelated`. Inherited per spec; default `auto`. Brush-level field so renderer backends can drive texture sampler state (URP `Texture.filterMode`, IMGUI `FilterMode`); `crisp-edges` aliases to `Pixelated` per common author intent. CSS spec values `smooth` and `high-quality` collapse to `Auto`. `Brush.Image(handle, rect, ImageRenderingMode)` factory carries it through; old `Brush.Image(handle, rect)` still works (defaults Auto). Tests: `ImageRenderingResolverTests` (10).

### Performance
- `CssStringUtil.ToLowerInvariantOrSame` — allocation-free `ToLowerInvariant` fast path that returns the SAME string instance when the input contains no ASCII uppercase chars. Wired through `StyleResolver.Display` / `WhiteSpace` / `TextAlign` (called per-element per-layout-pass) and `FlexProperties` keyword parsers. Cascade output is overwhelmingly already-lowercase (authors and the parser produce canonical form), so the fast path eliminates the per-call allocation on the hot layout path entirely. Tests added in `CssStringUtilTests` (now 21 total).
- `CssStringUtil` (`Runtime/Css/Values/CssStringUtil.cs`): allocation-free predicates `EqualsIgnoreCase`, `EqualsIgnoreCaseTrimmed`, `StartsWithIgnoreCase`, `IsCssColorFunctionPrefix`. Replace the `s.Trim().ToLowerInvariant() == "literal"` pattern that allocated two strings per call. Wired through `BoxShadowResolver`, `TextShadowResolver`, `ColorResolver`, `BackgroundResolver`, `BorderResolver`, `OverflowResolver`, `TransformResolver` — every paint-frame hot path that case-folded against CSS keywords. `CssNamedColors.TryGet` now uses `OrdinalIgnoreCase` dictionary lookup, eliminating the double-lowercasing on named-color resolution. Net: 12+ `ToLowerInvariant` call sites removed from the per-frame paint path. Tests: `CssStringUtilTests` (15).

- `::backdrop` pseudo-element painting for top-layer hosts (open modal `<dialog data-modal>` and open popovers `[popover][data-popover-open]`). `CascadeEngine` keeps `::backdrop` rules in a separate compile-time bucket and exposes `ComputeBackdrop(host)` (`Runtime/Css/Cascade/CascadeEngine.Backdrop.cs`); `BoxBuilder` / `SnapshotBoxBuilder` inject a synthetic backdrop sibling box before each top-layer host when `LayoutEngine.BackdropStyleOf` is wired (the golden runner wires it automatically). Backdrop ComputedStyle forces `position: fixed; top/right/bottom/left: 0; display: block; box-sizing: border-box` on top of the cascaded author rules so the box always covers the viewport. Inheritance is self-only per CSS Fullscreen — no host inheritance. v1 simplifications: top-layer ordering relies on `position: fixed` UA defaults rather than a true top-layer stacking pass; ancestors with transform/filter/will-change can disturb the containing block. Tests: `CascadeBackdropTests` (12), `TopLayerTests` (8), `BackdropBoxBuilderTests` (9). Golden snippet: `25-backdrop-modal-dialog`. Other pseudo-elements (`::before`/`::after`/`::placeholder`/`::selection`) remain unsupported in v1.
- `text-shadow` property (`Runtime/Paint/TextShadow.cs` + `Runtime/Paint/Conversion/TextShadowResolver.cs`) — per-glyph drop shadow with offset, blur-radius parsing, color, and comma-separated multiples. Inherited per CSS spec; default `none`. `BoxToPaintConverter.EmitTextRun` emits shadow `DrawTextCommand`s before the glyph layer; multiple shadows reverse-order so the first listed shadow paints nearest the glyph (per CSS Text Decoration §6). Shadow scratch buffer reused across calls (zero per-frame alloc once warm). v1 simplification: blur-radius is parsed and validated but `DrawTextCommand` has no blur channel — sharp offset shadows render correctly; blurred shadows fall back to crisp duplicates at the offset. Common HUD outline patterns (4-corner offset shadows, single drop shadow) are fully supported. Tests: `TextShadowResolverTests` (13), `TextShadowPaintTests` (6). Golden snippet: `26-text-shadow`.
- DevTools overlay (`Runtime/DevTools/`): F12-toggleable diagnostic IMGUI overlay rendered post-document. Four modes — Outlines (margin/border/padding/content rectangles, Chrome DevTools palette), DirtyTracking (red/amber/gray flash for layout/style/paint changes; decays over `FlashFrames`), Hover (`<tag.class#id>` header + W×H px @ X,Y + 10 most-relevant computed style props; routes through the document's `IHitTester`), Performance (FPS, frame ms, cascade/layout/paint ms via `Unity.Profiling.ProfilerRecorder` against `UIProfilerMarkers`, GC bytes/frame, paint cache hit ratio). Lives outside the main paint pipeline so it can't corrupt PaintList state. Editor mirror: `Window/Weva/DevTools` (`Editor/DevTools/DevToolsWindow.cs`). Tests: `DevToolsOverlayTests` (22 cases) + `DevToolsHotPathTests` (6 cases) covering enable/disable, each mode's emission, BoxOutlineRenderer's 4-rect-per-box contract, HoverInspector pointer resolution, CacheStats hit/miss accounting, and per-frame allocation bounds.
- Editor menu: `GameObject/Weva/New UIDocument` (`Editor/Templates/MenuTemplate.cs`) creates a scene-ready GameObject with `UIDocument` + `IMGUIDocumentRenderer` + `DevToolsOverlay`.
- Package README at `Packages/com.wevaui/README.md` — quick start, supported subset, architecture, v0.7 perf numbers, API surface, DevTools docs. ≤ 280 lines; deep dive lives in `PLAN.md`.
- Demo footer hint: the `Samples~/PhaseOneDemo` and `Assets/UI` menus now mention F12 for devtools alongside the existing hot-reload tip.
- Inline splitting per CSS Display Module Level 3 §2: a block-level descendant inside an inline element now forces the IFC to emit before/block/after segments instead of silently dropping the block descendant. `InlineLayout.CollectInline` records segment boundaries; `LayoutInline` runs the line-breaker per-segment and inserts the laid-out block as a sibling line-stack child between segments. The inline element's own decoration is not split between fragments (single-fragment-paint v1 simplification). Tests: `InlineSplittingTests` (11 cases). Golden snippet: `23-inline-splitting`.
- `text-overflow: ellipsis` post-pass in `EllipsisHelper`: when a container has clipping `overflow` (`hidden|scroll|clip|auto`), `white-space: nowrap`, and `text-overflow: ellipsis`, lines exceeding the container's content width are truncated in-place — the breaker walks runs left-to-right, finds the largest prefix whose width plus the ellipsis glyph fits, and replaces the trailing run with truncated-text + "…". Subsequent runs on the line are dropped. Multi-line ellipsis (`line-clamp`) is deferred to v2. Tests: `TextOverflowEllipsisTests` (9 cases). Golden snippet: `24-text-overflow-ellipsis`.
